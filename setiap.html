1. Kuantifikasi Proposisi

Dalam logika predikat, kita sering membuat pernyataan tentang semua anggota suatu himpunan atau tentang keberadaan setidaknya satu anggota. Dua jenis kuantifier utama adalah:

∀
Kuantifier Universal
"untuk setiap" / "semua"
∃
Kuantifier Eksistensial
"ada" / "terdapat" / "beberapa"

∀ — Kuantifier Universal

Pernyataan ∀x P(x) bernilai BENAR jika dan hanya jika P(x) bernilai benar untuk setiap elemen x di domain pembicaraan.
∀x P(x)  ≡  P(x₁) ∧ P(x₂) ∧ ··· ∧ P(xₙ)
Contoh 1. Domain: bilangan bulat positif.
P(x) = "x² ≥ 0"
∀x P(x) bernilai BENAR karena kuadrat setiap bilangan bulat ≥ 0.

Contoh 2. "Semua burung bisa terbang"
Misalkan domain = semua burung. Ini SALAH karena ada penguin (kontracontoh!).
Untuk membuktikan ∀x P(x) SALAH, cukup temukan satu kontracontoh (elemen x₀ di mana P(x₀) salah).

∃ — Kuantifier Eksistensial

Pernyataan ∃x P(x) bernilai BENAR jika dan hanya jika P(x) bernilai benar untuk setidaknya satu elemen x di domain.
∃x P(x)  ≡  P(x₁) ∨ P(x₂) ∨ ··· ∨ P(xₙ)
Contoh 1. Domain: bilangan bulat.
P(x) = "x² = 4"
∃x P(x) bernilai BENAR karena x = 2 (atau x = −2) memenuhi.

Contoh 2. "Ada bilangan prima yang genap"
BENAR: x = 2 adalah bilangan prima yang genap.
Untuk membuktikan ∃x P(x) SALAH, harus tunjukkan tidak ada satu pun elemen di domain yang memenuhi P(x).

Perbandingan Ringkas

NotasiDibacaBENAR jika…SALAH jika…
∀x P(x) Untuk setiap x, P(x) P(x) benar untuk SEMUA x Ada setidaknya SATU x yang membuat P(x) salah
∃x P(x) Terdapat x, P(x) Ada SETIDAKNYA SATU x yang membuat P(x) benar P(x) salah untuk SEMUA x di domain
∃! x P(x) Terdapat tepat satu x, P(x) Tepat SATU x yang memenuhi P(x) Nol atau lebih dari satu x yang memenuhi

Kuantifier Berlapis (Nested Quantifiers)

Jika dua kuantifier berbeda, urutan penulisan sangat berpengaruh pada makna!
PernyataanMaknaContoh (ℝ × ℝ)
∀x ∀y P(x,y)Semua pasangan (x,y) memenuhi Px+y = y+x → BENAR
∀x ∃y P(x,y)Untuk tiap x, ada y yang memenuhi P∀x ∃y (x+y=0) → BENAR (y=−x)
∃x ∀y P(x,y)Ada x tertentu yang untuk semua y memenuhi P∃x ∀y (x+y=0) → SALAH
∃x ∃y P(x,y)Ada setidaknya satu pasangan (x,y) yang memenuhix²+y²=1 → BENAR

2. Negasi Proposisi Berkuantifier

Hukum De Morgan untuk kuantifier menyatakan bahwa negasi dari pernyataan berkuantifier membalik kuantifier dan meniadakan predikat.

¬(∀x P(x))  ≡  ∃x (¬P(x))
¬(∃x P(x))  ≡  ∀x (¬P(x))
Hukum De Morgan untuk Kuantifier:
  • Negasi dari "SEMUA" menjadi "ADA YANG TIDAK"
  • Negasi dari "ADA" menjadi "SEMUA TIDAK"

Contoh Negasi

Pernyataan asli:
∀x (x² ≥ 0)
Artinya: "Semua bilangan kuadratnya ≥ 0"
Negasinya:
∃x ¬(x² ≥ 0)
Artinya: "Ada bilangan yang kuadratnya < 0"
Pernyataan asli:
∃x (x adalah prima ∧ x genap)
Artinya: "Ada prima yang genap"
Negasinya:
∀x ¬(prima(x) ∧ genap(x))
Artinya: "Semua prima tidak genap"
Pernyataan asli:
∀x (Manusia(x) → Fana(x))
Artinya: "Semua manusia adalah fana"
Negasinya:
∃x (Manusia(x) ∧ ¬Fana(x))
Artinya: "Ada manusia yang tidak fana"
Pernyataan asli:
∀x ∃y (x + y = 0)
Artinya: "Tiap x punya invers"
Negasinya:
∃x ∀y (x + y ≠ 0)
Artinya: "Ada x yang tidak punya invers"

3. Menentukan Nilai Kebenaran Kesimpulan

Sebuah argumen terdiri dari sekumpulan premis (pernyataan yang diasumsikan benar) dan sebuah kesimpulan. Argumen dikatakan valid jika: bilamana semua premis benar, maka kesimpulan pasti benar.

Argumen: P₁ ∧ P₂ ∧ ··· ∧ Pₙ → C  (tautologi)
Valid jika tidak mungkin semua premis benar dan kesimpulan salah.

Tiga Kondisi Argumen

KondisiPremis-premisKesimpulanArgumen
① Valid & Sehat Semua BENAR BENAR ✓ VALID & SOUND
② Valid tapi Tidak Sehat Ada yang SALAH — ✓ VALID (struktur logis benar)
③ Tidak Valid (Fallacy) Semua BENAR SALAH ✗ TIDAK VALID

Contoh Argumen Berkuantifier

Silogisme Universal (Modus Ponens Universal):

P1∀x (Manusia(x) → Fana(x))
P2Manusia(Socrates)
∴Fana(Socrates)
Struktur: ∀x (P(x)→Q(x)), P(a) ∴ Q(a) — VALID

Silogisme Eksistensial:

P1∃x (Mahasiswa(x) ∧ Pandai(x))
∴Ada mahasiswa yang pandai
Dari pernyataan eksistensial, kita bisa menyimpulkan keberadaan — VALID

Contoh Argumen Tidak Valid (Fallacy):

P1∀x (Burung(x) → BisaTerbang(x))
P2BisaTerbang(Lebah)
∴Burung(Lebah)
Ini adalah Fallacy of Affirming the Consequent. Dari P(a)→Q(a) dan Q(a), kita TIDAK bisa menyimpulkan P(a).

4. Aturan Inferensi Logika

Aturan inferensi adalah pola argumen valid yang sudah dikenal dan dapat langsung digunakan dalam pembuktian.

Aturan Inferensi Proposisional

NamaBentukContoh
Modus Ponens p, p→q ∴ q Hujan; Jika hujan maka banjir → Banjir
Modus Tollens ¬q, p→q ∴ ¬p Tidak banjir; Jika hujan maka banjir → Tidak hujan
Silogisme Hipotetis p→q, q→r ∴ p→r Belajar→Lulus; Lulus→Kerja → Belajar→Kerja
Silogisme Disjungtif p∨q, ¬p ∴ q Kucing atau Anjing; Bukan Kucing → Anjing
Penambahan (Addition) p ∴ p∨q Hujan → Hujan atau Angin
Penyederhanaan (Simplification) p∧q ∴ p Hujan dan Dingin → Hujan
Konjungsi (Conjunction) p, q ∴ p∧q Hujan; Dingin → Hujan dan Dingin
Resolusi p∨q, ¬p∨r ∴ q∨r A atau B; ¬A atau C → B atau C

Aturan Inferensi untuk Kuantifier

NamaBentukPenjelasan
Universal Instantiation (UI) ∀x P(x) ∴ P(c) Dari "semua x memenuhi P", pilih elemen spesifik c → P(c) benar
Universal Generalization (UG) P(c) (sembarang c) ∴ ∀x P(x) Jika P berlaku untuk elemen sembarang c (tidak khusus), simpulkan ∀x P(x)
Existential Instantiation (EI) ∃x P(x) ∴ P(c) (untuk c baru) Dari "ada x yang memenuhi P", beri nama c pada elemen tersebut
Existential Generalization (EG) P(c) ∴ ∃x P(x) Jika P(c) benar untuk elemen c tertentu, maka ∃x P(x) pasti benar

Contoh Pembuktian Bertahap

Premis:
P1: ∀x (Mahasiswa(x) → Belajar(x))
P2: ∀x (Belajar(x) → Lulus(x))
P3: Mahasiswa(Andi)
Buktikan: Lulus(Andi)
  1. Mahasiswa(Andi) → Belajar(Andi)  [UI dari P1, c = Andi]
  2. Belajar(Andi)  [Modus Ponens: langkah 1 + P3]
  3. Belajar(Andi) → Lulus(Andi)  [UI dari P2, c = Andi]
  4. Lulus(Andi)  [Modus Ponens: langkah 3 + langkah 2] ∴ QED

5. Simulator Argumen Logika

Gunakan simulator di bawah untuk memeriksa validitas berbagai jenis argumen logika secara interaktif.

🔵 Simulator A — Silogisme Universal

Periksa apakah argumen ∀x (P(x)→Q(x)), P(a) ∴ Q(a) valid.

🟡 Simulator B — Modus Ponens & Tollens

Masukkan dua premis dan kesimpulan untuk diperiksa.

🟢 Simulator C — Rantai Inferensi

Bangun rantai argumen dengan kuantifier dan periksa apakah kesimpulan dapat ditarik.
